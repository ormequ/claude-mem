# Project Root Override Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Make `CLAUDE_MEM_PROJECT_ROOT` a hard, shared project-identity override for every Claude Mem hook path.

**Architecture:** Keep project grouping in `getProjectContext`, the existing single identity boundary used by both SessionStart reads and session/observation writes. A non-empty environment value replaces the supplied cwd before project-name and worktree resolution; an unset or whitespace-only value leaves current behavior unchanged.

**Tech Stack:** TypeScript, Bun test, Node.js environment variables.

---

### Task 1: Add failing shared-root regression tests

**Files:**
- Modify: `tests/utils/project-name.test.ts:1-180`
- Test: `tests/utils/project-name.test.ts`

- [ ] **Step 1: Preserve `CLAUDE_MEM_PROJECT_ROOT` around each test**

Import `afterEach`, save the original environment value at module scope, and restore it after every test so a configured-root assertion cannot affect the normal-path or worktree tests.

```ts
const originalProjectRoot = process.env.CLAUDE_MEM_PROJECT_ROOT;

afterEach(() => {
  if (originalProjectRoot === undefined) {
    delete process.env.CLAUDE_MEM_PROJECT_ROOT;
  } else {
    process.env.CLAUDE_MEM_PROJECT_ROOT = originalProjectRoot;
  }
});
```

### Task 2: Implement the central identity override

**Files:**
- Modify: `src/utils/project-name.ts:78-100`
- Test: `tests/utils/project-name.test.ts`

- [ ] **Step 1: Resolve a non-empty configured root before cwd**

At the beginning of `getProjectContext`, trim `process.env.CLAUDE_MEM_PROJECT_ROOT`. When it is non-empty, derive its name using the existing `getProjectName` helper and return the shared non-worktree context immediately:

```ts
const configuredRoot = process.env.CLAUDE_MEM_PROJECT_ROOT?.trim();
if (configuredRoot) {
  const primary = getProjectName(configuredRoot);
  return { primary, parent: null, isWorktree: false, allProjects: [primary] };
}
```

Leave all existing cwd and worktree logic after that guard unchanged.

```bash

```

Expected: all project-name tests pass, including normal Git-root and worktree isolation cases with the environment restored between tests.
